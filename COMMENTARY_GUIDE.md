# Code Commentary Guide

The Python files now contain three kinds of documentation:

1. **Module docstrings** explain the purpose of each file.
2. **Class and function docstrings** explain reusable components.
3. **Inline comments** explain important decisions, relationships, queries, and workflow steps.

The comments focus on the reasoning behind the code rather than narrating every
individual line. This keeps the project readable while making it easier to
discuss during an interview or code review.
